# System Patterns

## 架构

当前项目采用无构建步骤的静态站点架构：

- `index.html`：页面结构与内容
- `styles.css`：视觉系统、布局、响应式样式
- `script.js`：轻量交互行为
- `resources/`：用户提供的物流图片素材

## 设计模式

- 单页落地页结构：导航、首屏、数据指标、服务、优势、资源展示、流程、联系 CTA、页脚
- 组件化 CSS class 命名，便于后续迁移到框架或拆分组件
- CSS 自定义属性集中管理品牌色、阴影、圆角与布局宽度

## 关键决策

- 暂不引入前端框架或包管理，降低预览和部署复杂度
- 图片直接从 `resources/` 引用，避免复制资源造成冗余
- 交互保持轻量：移动导航、滚动阴影、年份填充
